Rose n8n Agent System Tutorial

System Overview

The Rose n8n Agent System is an intelligent workflow automation platform that uses a network of specialized AI agents to handle various tasks across different services. The system is designed with a central orchestrator agent that routes requests to specialized subagents based on the task requirements.

Key Features

  • Multi-agent architecture with specialized agents for different services
  • Intelligent task routing and sequencing
  • Bidirectional communication between agents
  • Natural language processing of user requests
  • Integration with Slack, Asana, Gmail, and Google Calendar

Agent Architecture

The system uses a hub-and-spoke architecture with the Orchestrator Agent at the center:

Master Orchestrator Agent

Central coordinator that processes all incoming user requests and routes them to the appropriate specialized agent. It can sequence multi-step operations and manage the flow of information between agents.

Mail Agent

Handles email interactions including retrieving, filtering, and summarizing emails. Also converts emails into structured commands for Asana project/task creation.

Slack Agent

Manages all Slack operations including channel creation, user management, message retrieval and sending, conversation summarization, and channel archiving.

Asana Agent

Handles all Asana project and task management including creation, modification, listing, and deletion of projects and tasks.

Calendar Agent

Manages Google Calendar operations including creating, updating, searching, and retrieving events and meetings.

How the System Works

Request Flow

  1. Input Reception: System receives user input from Slack, Gmail, or Telegram
  2. Intent Recognition: Orchestrator analyzes the request to identify necessary agent(s)
  3. Task Routing: Request is sent to the appropriate specialized agent
  4. Task Execution: Specialized agent performs the required action
  5. Response Generation: Results are formatted and returned to the user

Multi-Step Operations

For complex requests requiring multiple agents, the Orchestrator sequences operations logically:

  1. Analyze request to identify all necessary subtasks
  2. Determine optimal sequence of operations
  3. Execute each step sequentially, passing outputs from one agent as inputs to the next
  4. Combine results into a coherent response

Example Scenarios

Scenario 1: Email to Asana Project Conversion

User Input: An email is received via Gmail trigger

Process:

  1. Orchestrator identifies email source and routes to Mail Agent
  2. Mail Agent converts email content into Asana command
  3. Orchestrator takes Mail Agent output and routes to Asana Agent
  4. Asana Agent creates project/tasks based on email content
  5. User receives confirmation of created project/tasks

Scenario 2: Slack Conversation to Asana Tasks

User Input: "Get messages from #project-planning, summarize them and create an Asana project with tasks"

Process:

  1. Orchestrator recognizes multi-step request involving Slack and Asana
  2. Slack Agent retrieves and summarizes channel messages
  3. Orchestrator passes summary to Asana Agent
  4. Asana Agent creates project with tasks based on summary
  5. User receives confirmation with project details

Usage Tips

  • Be specific in requests: Include clear action verbs and target platforms
  • Use platform-specific keywords: Include terms like "project," "task," "channel," "email," or "meeting"
  • For multi-step operations: Clearly outline the sequence you want ("First get X, then create Y")
  • For Slack operations: Always reference channels by name with the # prefix
  • For Asana operations: Be specific about project and task names

System Capabilities

Email Management

  • Retrieve, filter, count, and summarize emails
  • Convert email content into Asana tasks/projects
  • Search emails by sender, subject, or content

Slack Management

  • Create, archive, and unarchive channels
  • Add/invite users to channels
  • Retrieve and summarize channel messages
  • Send messages to channels or users

Asana Management

  • Create, modify, and delete projects
  • Add, update, and remove tasks
  • List all projects and tasks

Calendar Management

  • Create and schedule meetings and events
  • Search and retrieve calendar events
  • Update and delete events
